### 场景触发

- 用户需求涉及后台系统开发
- 需要定位接口代码位置

## 能力列表

| 能力     | 说明                       |
| -------- | -------------------------- |
| 系统识别 | 从需求中识别涉及的后台系统 |
| 环境检查 | 检查本地是否有相关系统仓库 |
| 自动克隆 | 从 Git 仓库克隆缺失的系统  |
| 开发引导 | 引导选择参照接口进行开发   |
| 代码审查 | 检查代码是否符合规范       |

## 工作流程

```
用户需求
   ↓
阶段 1: 需求分析 → 识别涉及系统 + 开发类型
   ↓
阶段 2: 环境检查 → 读取/询问本地目录 → 检查系统是否存在
   ↓
阶段 3: 自动克隆 → Git clone 缺失系统
   ↓
阶段 4: 开发引导 → 新增接口/编辑接口 → 生成指引
   ↓
阶段 5: 代码审查 → 检查代码风格 + 规范
   ↓
阶段 6: 提交建议 → 生成 commit message
```

## 配置示例

### 首次使用

```
用户：我要在 abs-go 新增一个回放下载接口

Agent:
1. 识别需求：系统=abs-go, 类型=新增接口
2. 📁 检测到您是首次使用，请提供本地后台系统目录
   （例如：/Users/liuwei/Projects/alive-systems）
3. 用户提供后保存到 .env
4. 继续执行环境检查和开发引导
```

### 后续使用

```
用户：修改 alive-content 的配置接口

Agent:
1. 从 .env 读取 ALIVE_SYSTEMS_BASE_DIR
2. 检查 alive-content 目录是否存在
3. 如缺失，询问是否克隆
4. 存在则直接引导开发
```

## 错误处理

| 错误场景     | 处理策略           |
| ------------ | ------------------ |
| 目录不存在   | 提示确认或创建     |
| Git 失败     | 检查网络/权限      |
| 系统不在清单 | 提示确认或手动添加 |
| 文档无法访问 | 请求用户提供       |
