# 设计原则

---

## 7 条原则

### 1. 一致性

同语义元素在跨页面、跨端保持命名、样式、交互一致。同层级信息使用一致的排版层级和间距。同类状态颜色不混用。

### 2. 可预见性

危险操作前提供确认，说明影响范围与不可逆后果。长耗时任务提供进度或阶段性状态。按钮文案表达结果导向（如"发布课程""确认删除"）。

### 3. 效率优先

高频路径优先单屏完成。列表场景支持筛选、批量操作、快速编辑。复用最近选择、默认值、模板化配置。

### 4. 反馈及时

| 层级 | 场景 | 推荐组件 |
|------|------|---------|
| 即时轻反馈 | 保存成功、复制完成 | Message / Toast |
| 上下文反馈 | 表单字段校验、列表空态 | Inline Error / Empty |
| 中断反馈 | 危险确认、关键失败 | Dialog / Modal |
| 持续反馈 | 后台任务、审批进度 | Notification / 状态条 |

不允许无反馈提交。错误提示必须包含原因与下一步建议。

### 5. 渐进披露

首屏展示决策必需信息，扩展信息收进"更多/高级设置/折叠区"。复杂流程拆分为步骤或分组。

### 6. 可恢复性

支持撤销的动作优先提供"撤销"入口。不可逆动作必须二次确认。网络失败时提供重试入口。

### 7. 可访问性

文本对比度 ≥ 4.5:1（正常文字），≥ 3:1（大字 & UI 组件）。关键控件支持键盘访问与语义标签。

---

## Do

- 使用 Design Token 值，不硬编码
- 使用注册组件，不新造基础组件名
- 管理台列表页用表格，不用卡片网格
- 每个容器最多一个主按钮
- B 端用"您"，C 端用"你"
- 每个组件覆盖 loading / empty / error / disabled / hover(focus) 状态
- 文字对比度 ≥ 4.5:1
- ESC 关闭 Dialog 和 Drawer
- 所有交互元素支持键盘访问，有可见 focus ring
- 优先选模板再选组件（template-first, then component）

## Don't

- 不跳过产品逻辑和终端判断直接设计
- 不使用外部设计系统的值替代 XEDS
- 不生成"看起来像别的公司的后台"的页面
- 不使用毛玻璃、重渐变、发光效果、大面积装饰阴影
- 不编造业务事实、组件名、token 值 — 用 `待确认`
- 不在代码中使用裸 hex 色值 — 用 CSS 变量
- 不使用 `--color-info` — 它不存在
- 不混用 Popup / ActionSheet / Modal / Toast 语义
- 移动端触控区域不低于 44px
- 不使用暖色调阴影，仅 `rgba(0,0,0, 0.05~0.16)`

## 页面级检查清单

| 页面类型 | 必须满足 |
|---------|---------|
| 列表页 | 筛选可恢复、批量操作可预览影响、空态可引导 |
| 详情页 | 信息分组明确、操作反馈即时、返回后上下文保留 |
| 表单页 | 必填与错误可见、提交前风险提示、提交后状态明确 |
| 数据页 | 指标口径一致、时间范围可切换、异常数据有解释 |
| 设置页 | 变更影响可说明、危险配置二次确认、可回滚 |

## 一句话自检

> 这个页面首先看起来像小鹅通现有系统里的有效页面，而不是某个外部产品的精致模仿？

如果答案不是"是"，停止装饰，回到规范重新对照。
